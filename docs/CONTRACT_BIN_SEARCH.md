# Динамический поиск организации по БИН

## Описание функционала

При создании договора в системе клиники реализован динамический поиск организации работодателя по БИН.

## Как это работает

1. **Автоматический поиск**: Когда пользователь вводит 12-значный БИН, система автоматически начинает поиск через 500мс после последнего ввода символа (debounce).

2. **Визуальная индикация**:
   - **Поиск в процессе**: Показывается анимированный спиннер и сообщение "Поиск организации..."
   - **Организация найдена**: Зеленая галочка и сообщение с названием организации
   - **Организация не найдена**: Красный крестик и предупреждение о том, что организация не зарегистрирована

3. **Цветовая индикация поля**:
   - Зеленая рамка - организация найдена
   - Красная рамка - организация не найдена
   - Обычная рамка - поиск не выполнялся или БИН неполный

## Сообщения пользователю

### Организация найдена
```
✓ Найдена организация: [Название организации]
```

### Организация не найдена
```
✗ Данная организация не зарегистрирована в системе. 
Договор будет создан, и работодатель получит уведомление для регистрации.
```

## Технические детали

- **Файл**: `frontend/app/dashboard/clinic/contracts/page.tsx`
- **Функция поиска**: `handleSearchBin(bin: string)`
- **Debounce**: 500мс
- **Минимальная длина БИН**: 12 символов
- **API метод**: `workflowStoreAPI.findEmployerByBin(bin)`

## Преимущества

1. Мгновенная обратная связь пользователю
2. Автоматическое заполнение телефона, если организация найдена
3. Предупреждение о незарегистрированных организациях
4. Улучшенный UX с визуальными индикаторами
5. Снижение ошибок при вводе данных
